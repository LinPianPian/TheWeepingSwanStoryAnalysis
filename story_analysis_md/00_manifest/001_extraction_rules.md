# 抽取与拆解规则

## 场景切分
场景主要按以下条件切分：
- 地点变化
- 时间变化
- 叙事模式变化（记 / 忆 / 梦 / 幻觉 / 红楼尾声）
- 对话对象或冲突核心变化
- 方知宥认知状态变化（想起、误认、病发、真相被改写）

## 对白抽取
- 先按 `角色名：[台词]` 形式逐行抽出。
- 只要原文有不同 speaker label，就先当作不同标签保存。
- 后续是否归并为同一人物，要看 `02_characters/000_character_identity_registry.md`。

## 选项残留标记
- 所有以 `*` 开头的行都进入索引。
- `结局选择开始` 与 `【结局：红楼】` 也视作关键结构标记。
- 这些标记在本项目中不当作多分支剧情树，而当作单线文本中的残留结构痕迹。

## 证据口径
- 文本直述：原文明确说出
- 强烈暗示：原文未直说，但上下文强支持
- 解释性推断：基于多处证据归纳，不冒充原文事实

## 引证格式
统一采用：`原文依据：story.txt:起始行-结束行`
